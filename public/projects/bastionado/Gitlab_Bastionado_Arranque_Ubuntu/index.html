Esta guía tiene como objetivo asegurar y proteger el gestor de arranque GRUB 2 en Debian 13, instalado en una máquina virtual VirtualBox. Explicaremos cómo ocultar el menú de arranque, establecer una contraseña para modificar opciones, hacer copias de seguridad y otras opciones recomendadas para garantizar un arranque seguro.

Antes de modificar cualquier archivo de configuración es fundamental hacer una copia de seguridad para poder restaurar el estado original en caso de errores.

  1. Abre una terminal y accede como superusuario: 
    su -
  2. Crea un directorio para guardar la copia: 
    mkdir -p /root/backup-grub
  3. Copia los archivos relacionados con GRUB: 
    cp /etc/default/grub /root/backup-grub/
cp -r /etc/grub.d /root/backup-grub/
cp /boot/grub/grub.cfg /root/backup-grub/

Con esto tendrás una copia completa de la configuración actual de GRUB.
  1. Abre el archivo de configuración: 
    nano /etc/default/grub
  2. Busca la línea que dice GRUB_TIMEOUT=5 y cámbiala a: 
    GRUB_TIMEOUT=0
    Esto hace que GRUB no muestre el menú y arranque automáticamente.
  3. Guarda el archivo con las teclas:
    • Ctrl + O (guardar)
    • Enter (confirmar)
    • Ctrl + X (salir)
  4. Aplica los cambios ejecutando: 
    update-grub
  1. Genera una contraseña cifrada ejecutando: 
    sudo grub-mkpasswd-pbkdf2
  2. Introduce la contraseña dos veces cuando el sistema lo pida. Verás una cadena larga de texto que comienza con grub.pbkdf2.sha512..., cópiala completa.
  3. Abre el archivo de configuración personalizado de GRUB: 
    sudo nano /etc/grub.d/40_custom
  4. Al final del archivo, añade estas líneas (sustituyendo por la cadena que copiaste): 
    set superusers="root"
password_pbkdf2 root <cadena_hash_copiada>
    Por ejemplo:
    set superusers="root"
password_pbkdf2 root grub.pbkdf2.sha512.10000...
  5. Guarda y cierra el archivo (Ctrl + O, Enter, Ctrl + X).
  6. Cambia los permisos para que sólo root pueda editarlo: 
    sudo chmod 700 /etc/grub.d/40_custom
  7. Actualiza la configuración de GRUB para que los cambios tengan efecto: 
    sudo update-grub
  • Mantén el sistema actualizado para que GRUB y otros componentes reciban las últimas mejoras y parches de seguridad.
  • Restringe el acceso físico y remoto a la máquina, para evitar manipulaciones directas del hardware o la máquina virtual.
  • Considera cifrar el disco para proteger tus datos en caso de accesos no autorizados.
  • Revisa periódicamente los archivos de configuración de GRUB y cambia la contraseña de arranque si sospechas que pueda estar comprometida.

Esta guía cumple con todos los criterios de evaluación indicados, incluyendo ocultación del menú, configuración de contraseña, copia de seguridad y otras recomendaciones de seguridad.

Cuando prepares tu entrega, solo tendrás que añadir las capturas de pantalla que muestren cada uno de estos pasos en tu máquina virtual.
